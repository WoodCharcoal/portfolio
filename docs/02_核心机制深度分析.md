# 核心机制深度分析

本文件将核心机制拆分为可验证的设计面向，并给出玩法示例与设计取舍。

1. 时间属性（Temporal Tag）
- 定义与行为：FLUID 与 ANCHORED 的差异性明确，便于玩家建立心智模型。
- 可视化要点：颜色区分（红/蓝）、粒子与回溯动画提示、状态图标

2. 回溯卡牌拆解
- 全局回溯：优点为直觉易用；缺点为粗暴，需配合锚定机制
- 碎片回溯：优点为精确可控；成本为玩家在空间上需额外规划
- 信标回溯：提供“保护”的反向控制，带来最高的策略深度

3. 交互细节与边界情况
- 信标为 FLUID：当信标自身被回溯，需要处理保护场是否随之恢复或消失
- 同格冲突：回溯后位置冲突判定（优先锚定体，再为流动体寻找邻近可用格）
- 连续回溯：多次回溯叠加下的状态一致性校验

4. 数值与资源设计
- 回溯次数分配：教学期多（1-3），练习期（2-3），挑战期（≤2）
- 信标半径数值建议：2~3（以 9x9 网格为基准）

5. 可玩性验证（测试假设）
- 假设 A：回溯能将“死局”转化为“教学点”。验证方式：AB 测试对比无回溯/有回溯的玩家通关率与 Aha 时刻出现频率
- 假设 B：碎片回溯 + 信标能产生更丰富的解法树。验证方式：统计每关不同解法数量的分布
