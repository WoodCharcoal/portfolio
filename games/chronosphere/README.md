# 时空回溯 (Chronosphere)

一款在经典推箱子（Sokoban）玩法基础上，引入时间操控机制的单人解谜游戏。

- **平台**: 移动端 + PC (TapTap平台)
- **类型**: 解谜游戏（推箱子变体）
- **状态**: 已上线 (v1.0.37)
- **试玩**: https://www.taptap.cn/app/834934

## 核心特色

### 机制创新
- **时间属性二元对立**: 可回溯 vs 锚定 的核心设计
- **三大类回溯系统**: 全局回溯 → 碎片回溯 → 信标回溯
- **递进式组合**: 从单一机制到多类型组合的玩法深度

### 教学设计
- **20关教学曲线**: 严格遵循"引入→练习→组合→挑战"节奏（未完成）
- **约束创造发现**: 通过地形设计迫使玩家发现机制
- **渐进放权**: 前期结构简化放宽容错率，后期资源收紧，要求精确步数完成关卡

### 系统设计
- **Hub World**: 用核心玩法作为导航方式
- **关卡编辑器**: 提供组件介绍，降低UGC创作门槛
- **社区系统**: 完整的创作者→社区星级评价系统→玩家反馈循环

## 设计方法论

```
设计假说 → 机制验证 → 教学曲线 → 关卡实现 → 玩家测试 → 迭代优化
```

每个设计决策都基于：
- **玩家心理模型**: 玩家会如何理解这个机制？
- **认知负荷**: 是否超过学习容量？
- **容错空间**: 是否允许试错？
- **反馈清晰度**: 玩家能否快速理解结果？

## 设计数据

- **关卡数量**: 20关（官方）+ 社区无限
- **机制类型**: 3大类（全局回溯、碎片回溯、信标保护）
- **时间属性**: 2种（FLUID可回溯、ANCHORED不可回溯）
- **难度阶段**: 4个（引入→练习→组合→挑战）
