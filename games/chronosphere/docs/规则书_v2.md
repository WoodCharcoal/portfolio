# 《时空回溯》规则书 v2.0

> 本文档是纯粹的游戏机制规则定义，不涉及 UI 交互或视觉表现。

---

## 第一章：棋盘与基础概念

### 1.1 棋盘

- 棋盘由 W×H 网格单元格组成，尺寸范围 5×5 至 20×20
- 坐标系：原点 (1,1) 位于左上角，X 向右递增，Y 向下递增
- 所有物体占据一个完整格子，坐标为正整数

### 1.2 格子类型

| 类型 | 标识 | 说明 |
|------|------|------|
| 空地 | EMPTY | 可自由通行 |
| 墙壁 | WALL | 不可通行，不可破坏 |
| 目标点 | TARGET | 可通行，胜利条件标记 |
| 开关 | SWITCH | 可通行，踩下触发信号 |
| 门 | DOOR | 根据信号开/关，关闭时不可通行 |

### 1.3 物体类型

| 类型 | 标识 | 可移动 | 说明 |
|------|------|--------|------|
| 玩家 | PLAYER | 主动移动 | 唯一控制单位 |
| 箱子 | BOX | 被推动 | 核心谜题元素 |
| 信标 | BEACON | 被推动 | 提供保护场 |

### 1.4 时空标签

每个物体具有一个时空标签，决定其是否受回溯影响：

| 标签 | 含义 | 回溯表现 |
|------|------|---------|
| TEMPORAL_FLUID（流动体） | 随时间流动 | 受回溯影响，可被回溯到初始位置 |
| TEMPORAL_ANCHORED（锚定体） | 锚定在时间线上 | 不受任何回溯影响 |

**标签派生规则**：
- 玩家：prop=1 → 流动体，prop=0 → 锚定体（默认锚定）
- 箱子：prop=1 → 流动体（幻影箱），prop=0 → 锚定体（锚定箱）
- 信标：prop 为奇数 → 流动体，prop 为偶数 → 锚定体

## 第二章：移动与推动

### 2.1 玩家移动

- 玩家每次向四个正交方向之一移动一格（上/下/左/右）
- 不可斜向移动
- 每次有效移动计为一步

**移动阻止条件**（任一满足则移动失败）：
1. 目标格子超出棋盘边界
2. 目标格子为墙壁
3. 目标格子为关闭的门
4. 目标格子被物体占据且该物体不可推动（见 §2.2）

### 2.2 推动规则

当玩家目标格子有可推动物体（箱子或信标）时，触发推动判定：

**推动执行条件**（全部满足时推动成功）：
1. 被推物体后方格子在棋盘范围内
2. 被推物体后方格子非墙壁/非关闭的门
3. 被推物体后方格子无其他物体

**推动约束**：
- 推动方向与玩家移动方向一致
- 每次只能推动一个物体（不可链式推动）
- 只能推不能拉
- 推动成功时玩家和被推物体同时移动

### 2.3 移动历史

- 每步移动记录所有参与的物体移动信息（玩家移动 + 可能的推动）
- 历史栈最多保留 50 步，超出时自动丢弃最早记录

## 第三章：信号系统（开关与门）

### 3.1 开关

- 开关是一种特殊格子类型，物体可站在其上
- 当匹配的物体位于开关格子时，开关被触发
- 每个开关有以下属性：
  - **channels**（频道 bitmask）：触发时激活哪些频道
  - **acceptTypes**：哪些物体类型可触发（默认：玩家/箱子/信标均可）
  - **inverted**：反转模式，未触发时激活信号

### 3.2 门

- 门是一种特殊格子类型，有开/关两种状态
- 关闭时阻挡所有通行，开启时等同空地
- 每个门有以下属性：
  - **channels**（频道 bitmask）：监听哪些频道
  - **condition**：`"any"`（任一频道有信号即开）或 `"all"`（所有频道都有信号才开）
  - **inverted**：反转开关状态

### 3.3 信号计算

每次物体移动后重新计算信号状态：
1. 遍历所有开关，判定是否被触发
2. 将触发的开关频道合并为全局活跃信号 bitmask
3. 每个门根据自身 channels 和 condition 判定开/关

## 第四章：锚定系统

### 4.1 永久锚定

- 由物体的时空标签（TEMPORAL_ANCHORED）决定
- 在游戏过程中不可改变
- 永久锚定物体完全不受任何回溯机制影响

### 4.2 临时锚定（信标保护场）

**保护场定义**：
- 以信标为中心的曼哈顿距离区域
- 保护半径由信标的 prop 值决定：`radius = floor(prop / 2)`
- 判定公式：`|物体X - 信标X| + |物体Y - 信标Y| ≤ radius`

**保护效果**：
- 保护场内的**流动体**箱子获得临时锚定，不受回溯影响
- 保护场内的**流动体**信标获得临时锚定（需排除自身保护场，仅由其他信标保护）
- 临时锚定实时计算：物体进入保护场即获得，离开即失去

### 4.3 锚定优先级

回溯判定时按以下优先级检查（从高到低）：
1. **永久锚定** → 不回溯
2. **信标保护场内** → 不回溯
3. **流动体且不在保护场内** → 参与回溯

## 第五章：回溯系统

### 5.1 回溯总则

- 回溯将受影响的物体恢复到**初始位置**（关卡起始位置）
- 回溯操作本身计为一步，压入历史栈（可被撤销）
- 回溯只在静止状态下执行（非动画中、非已通关）

### 5.2 三种回溯类型

#### 全局回溯（Global）

**作用范围**：棋盘上所有物体

**执行逻辑**：
1. 遍历所有物体，跳过永久锚定物体
2. 跳过处于信标保护场内的物体
3. 将剩余流动体恢复到初始位置

**模式变体**：
- `fullReset = true`：标准全局回溯（所有非保护物体回到初始位置）
- `fullReset = false`：限步全局回溯（仅回溯最近 N 步涉及的物体）

#### 碎片回溯（Fragmented）

**作用范围**：指定区域内的物体

**区域参数**：
- `centerGx, centerGy`：区域中心坐标
- `rangeMode`：`"manhattan"`（菱形）或 `"square"`（方形）
- `rangeSize`：菱形时为半径，方形时为边长

**执行逻辑**：
1. 确定区域范围内的所有物体
2. 跳过永久锚定物体
3. 跳过处于信标保护场内的物体
4. 将剩余流动体恢复到初始位置
5. 从历史栈中移除被回溯物体的相关移动记录

**无区域时的退化行为**（旧兼容）：
- 回溯最近 N 步涉及的所有物体（N 由关卡配置或卡牌指定）

#### 信标回溯（Beacon）

**作用范围**：以信标保护场为分割线

**执行流程**：
1. 记录所有物体当前位置
2. 将所有流动体恢复到初始位置
3. 以初始位置的信标重新计算保护场
4. 回溯前位于保护场内的物体，恢复到其当前位置（不回溯）
5. 不在保护场内的物体保持初始位置（已回溯）

**目标模式**：
- `targetMode = "all"`：使用所有信标的保护场
- `targetMode = "single"`：仅使用指定信标的保护场

**核心语义**：保护场外归零，保护场内保留。

### 5.3 两阶段执行原则

所有回溯类型共享的关键实现原则：

> **先判定，后执行** — 先用当前状态统一判定哪些物体应被回溯，再统一执行位置恢复。避免先回溯的物体（如信标）改变位置后影响后续物体的保护场判定。

### 5.4 回溯与历史的关系

| 回溯类型 | 对历史栈的影响 |
|---------|--------------|
| 全局（fullReset） | 回溯操作压入一步 |
| 全局（限步） | 移除最近 N 步，回溯操作压入一步 |
| 碎片（区域） | 移除被回溯物体的相关记录，回溯操作压入一步 |
| 碎片（按步数） | 移除最近 N 步，回溯操作压入一步 |
| 信标 | 回溯操作压入一步 |

## 第六章：撤销系统（Undo）

### 6.1 基本撤销

- 撤销回退历史栈中最近一步
- 将该步涉及的所有物体恢复到移动前位置
- 步数计数 -1

### 6.2 信标保护对撤销的影响

撤销时同样受信标保护场约束：
- 当前处于信标保护场内的物体**不参与撤销**
- 可回溯信标若在其他信标保护场内，同样不参与撤销
- 玩家不受信标保护场影响，始终可撤销

## 第七章：卡牌系统

### 7.1 卡牌定义

回溯操作通过卡牌触发。每张卡牌定义：

| 属性 | 说明 |
|------|------|
| type | 卡牌类型：`"global"` / `"fragmented"` / `"beacon"` |
| uses | 剩余使用次数 |
| maxUses | 最大使用次数 |
| fullReset | 是否完全重置（true=回到初始位置，false=按步数回退） |
| steps | 非完全重置时的回退步数 |
| rangeMode | 碎片卡牌的区域形状：`"manhattan"` / `"square"` |
| rangeSize | 碎片卡牌的区域大小 |
| targetMode | 信标卡牌的目标模式：`"all"` / `"single"` |

### 7.2 卡牌消耗

- 使用卡牌时 uses - 1
- uses = 0 时卡牌不可用
- 关卡重置时卡牌恢复到初始状态

### 7.3 关卡配置

每个关卡可配置若干张卡牌，关卡设计者决定：
- 给什么类型的卡牌
- 每张卡牌的使用次数
- 卡牌的具体参数（区域大小、步数等）

## 第八章：胜利条件

### 8.1 标准胜利条件

当且仅当**所有目标点都被可接受的物体覆盖**时，判定为胜利。

### 8.2 目标点的可接受类型

每个目标点可独立配置 `acceptTypes`：

| 配置 | 可接受物体 |
|------|-----------|
| 未配置（默认） | 任何非玩家物体（箱子、信标） |
| `{ box = true }` | 仅箱子 |
| `{ player = true }` | 仅玩家 |
| `{ beacon = true }` | 仅信标 |
| 任意组合 | 组合中的类型均可 |

### 8.3 胜利判定时机

以下事件后自动触发胜利检查：
- 玩家移动完成
- 推动完成
- 回溯完成

### 8.4 胜利效果

- 胜利后冻结所有输入（不可再移动）
- 胜利后不可执行回溯操作

## 第九章：碰撞规则

### 9.1 碰撞体定义

以下物体为碰撞体（同一格子最多一个碰撞体）：
- 玩家
- 箱子
- 信标

以下不参与碰撞：
- 目标点（可与碰撞体重叠）
- 开关（可与碰撞体重叠）
- 空地、墙壁、门（由格子类型决定通行性）

### 9.2 碰撞判定

- 物体不可移动到已有碰撞体的格子（推动情况除外）
- 推动时被推物体不可进入已有碰撞体的格子
- 碰撞发生时移动无效（无任何物体位移）

---

## 第十章：关卡约束

### 10.1 必要条件

- 必须有且仅有一个玩家
- 必须有至少一个目标点
- 玩家初始位置不可与其他碰撞体重叠

### 10.2 推荐约束

| 约束项 | 推荐值 |
|--------|--------|
| 信标数量 | ≤ 10 |
| 箱子总数 | ≤ 50 |
| 棋盘面积 | ≤ 400 格 (20×20) |

## 附录 B：回溯判定流程图

```
物体 X 是否应被回溯？
│
├─ X 是永久锚定体？ ──→ YES → 不回溯
│
├─ X 是信标？
│   └─ X 在其他信标保护场内？ ──→ YES → 不回溯
│
├─ X 是非信标物体？
│   └─ X 在任意信标保护场内？ ──→ YES → 不回溯
│
└─ X 当前位置 ≠ 初始位置？ ──→ YES → 回溯到初始位置
                             └─ NO  → 无需回溯（已在原位）
```

信标回溯执行：

│
