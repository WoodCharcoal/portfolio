# 系统设计文档

本文档覆盖 Hub World、关卡编辑器与社区系统的设计要点与交互流程。

1. Hub World
- 结构：走廊 + 凹槽（关卡入口）+ 社区/编辑器入口
- 交互：将对应方块推入凹槽即进入关卡
- 设计目的：交互一致性、空间叙事、功能发现

2. 关卡编辑器
- 核心功能：绘制地形、放置物体、配置目标、设置回溯卡牌、实时预览
- 存档与分享：6 个本地存档位；必须通关才能保存并上传
- 格式：使用 JSON 序列化关卡数据，支持导入/导出

3. 社区系统
- 流程：创作→通关验证→上传→浏览/下载→评分/反馈→迭代
- 推荐机制：根据完成人数/点赞/难度曲线自动推荐“精选”关卡
- 反作弊：上传前强制通关，检测无限循环/无解结构

4. 技术实现建议
- 回溯实现：场景初始快照 + 标签过滤回放
- 保护场判定：曼哈顿距离实时计算
- 网络：关卡 JSON 存储 + CDN 分发，支持离线编辑与云同步
