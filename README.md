## 📚 项目总览

### 《时空回溯》(Chronosphere)
一款在经典推箱子（Sokoban）玩法基础上，引入时间操控机制的单人解谜游戏。

- **平台**: 移动端 + PC (TapTap)
- **类型**: 解谜游戏（推箱子变体）
- **状态**: 已上线 (v1.0.32)
- **特点**: 核心机制创新、教学曲线设计、UGC编辑器

## 🎮 项目亮点

### 1️⃣ 机制创新
- **时间属性二元对立**: FLUID vs ANCHORED 的核心设计
- **三层回溯系统**: 全局回溯 → 碎片回溯 → 信标保护
- **递进式组合**: 从单一机制到多维组合的玩法深度

### 2️⃣ 教学设计
- **20关教学曲线**: 严格遵循"引入→练习→组合→挑战"节奏
- **约束创造发现**: 通过地形设计迫使玩家发现机制
- **渐进放权**: 前期资源充足，后期逐步收紧

### 3️⃣ 系统设计
- **Hub World**: 用核心玩法作为导航方式
- **关卡编辑器**: 降低UGC创作门槛
- **社区系统**: 完整的创作者→玩家反馈循环

## 📊 设计方法论

### 核心设计流程

```
设计假说 → 机制验证 → 教学曲线 → 关卡实现 → 玩家测试 → 迭代优化
```

### 关键设计决策

每个设计决策都基于：
- **玩家心理模型**: 玩家会如何理解这个机制？
- **认知负荷**: 是否超过学习容量？
- **容错空间**: 是否允许试错？
- **反馈清晰度**: 玩家能否快速理解结果？

## 📈 设计数据

- **关卡数量**: 20关（官方）+ 社区无限
- **机制类型**: 3种（全局回溯、碎片回溯、信标保护）
- **时间属性**: 2种（FLUID、ANCHORED）
- **难度阶段**: 4个（引入→练习→组合→挑战）
- **教学关卡**: 12关
- **挑战关卡**: 5关
