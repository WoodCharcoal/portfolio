## 🎮 项目亮点

### 1️⃣ 机制创新
- **时间属性二元对立**: 可回溯 vs 与锚定 的核心设计
- **三大类回溯系统**: 全局回溯 → 碎片回溯 → 信标回溯
- **递进式组合**: 从单一机制到多类型组合的玩法深度

### 2️⃣ 教学设计
- **20关教学曲线**: 严格遵循"引入→练习→组合→挑战"节奏（未完成）
- **约束创造发现**: 通过地形设计迫使玩家发现机制
- **渐进放权**: 前期结构简化放宽容错率，后期资源收紧，要求精确步数完成关卡

### 3️⃣ 系统设计
- **Hub World**: 用核心玩法作为导航方式
- **关卡编辑器**: 提供组件介绍，降低UGC创作门槛
- **社区系统**: 完整的创作者→社区星级评价系统→玩家反馈循环

## 📊 设计方法论

### 核心设计流程

```
设计假说 → 机制验证 → 教学曲线 → 关卡实现 → 玩家测试 → 迭代优化
```

### 关键设计决策

每个设计决策都基于：
- **玩家心理模型**: 玩家会如何理解这个机制？
- **认知负荷**: 是否超过学习容量？
- **容错空间**: 是否允许试错？
- **反馈清晰度**: 玩家能否快速理解结果？

## 🔍 核心设计问题

这个项目的设计过程围绕三个核心问题展开：

| 问题 | 解答 | 文档 |
|------|------|------|
| 如何让"回溯"为解题的必要条件而非可选便利？ | 通过时间属性二元对立和约束设计 | [核心机制](docs/02_核心机制深度分析.md) |
| 如何让不同类型的回溯产生组合深度？ | 三层递进系统 + 组合矩阵 | [核心机制](docs/02_核心机制深度分析.md) |
| 如何让玩家逐步理解并掌握这些能力？ | 严格的教学曲线与约束创造发现 | [关卡设计](docs/03_关卡设计哲学.md) |
